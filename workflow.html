阶段1 调研
1调研阶段
/module-kickoff
方法（5 步顺序执行，每步确认后再进下一步）
  • 竞品对标：选 5-6 家竞品，5 维度扫描（功能覆盖 / UI交互 / 差异化 / 用户评价 / looply启示）
  • 合规扫描：联邦→州级→平台政策→行业惯例，区分"必须遵守"和"建议遵守"
  • 服务商选型：识别外部依赖，每个 2-3 候选，功能/价格/接入复杂度/市场适配度对比
  • 需求优先级：汇总功能点，RICE 框架评估，划分 MVP / V2 / V3
  • 模块启动报告：前 4 步关键结论浓缩成一页纸
产出
  • 竞品对标.md
  • 竞品UI分析.md
  • 合规扫描.md
  • 服务商选型.md
  • 需求优先级.md
  • 模块启动报告.md

6PRD
/prd-writing
方法
  • 动手前交叉对齐所有输入源（流程图 / 设计稿 / ER图 / 架构图），列出覆盖差异
  • 按页面/功能逐章撰写，每章关联 PC + H5 设计稿
  • 写完后路径完整性检查 + 设计稿交叉校验
产出
  • Word (.docx) 带自动目录
  • Markdown (.md) 在线阅读版

阶段4 评审
7评审与反馈
/prd-review-simulation   /dev-feedback
方法
  • 模拟开发 / 测试多角色视角审查 PRD，问题按 P0-P3 分级
  • 修订 PRD 后再跑一轮验证，直到无 P0/P1
  • 开发反馈逐条确认，区分需求澄清 / 设计变更 / 技术建议
  • 同步修改 PRD + 设计稿，保持两边一致
产出
  • 模拟评审问题清单（P0-P3 分级）
  • PRD 修订版本
  • 开发反馈处理记录 + 回复草稿
