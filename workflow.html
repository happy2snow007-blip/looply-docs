实体关系图
/er-diagram-generation
工作方法
  • 从架构图提取实体，逐实体定义字段、类型、约束、枚举
  • Python 数据驱动 + Graphviz 生成 SVG，内嵌 JSON + YAML 双 metadata 供开发提取
  • 12 条自检清单分三阶段执行（实体设计 → 字段设计 → 整体校验）
产出
  • SVG 实体关系图（含 JSON + YAML 双 metadata）
  • Python 生成脚本（唯一数据源，迭代只改数据区）
沉淀规范
  • ER图 skill~/.claude/skills/er-diagram-generation/SKILL.md
  • 生成脚本模板~/.claude/skills/er-diagram-generation/gen_er_template.py
前置依赖
  • brew install graphviz — Graphviz 命令行工具（dot 引擎）
  • pip3 install graphviz pyyaml — Python 库

3 版本交付
需求迭代版本管理
/req-versioning
推荐主入口
  • 统一覆盖日常持续记录与交付前收口，优先替代分开判断旧入口
  • 模式 A：修改 PRD、原型、架构图、ER 图、流程图后持续记录进行中版本
  • 模式 B：准备交付开发时，基于前后交付文件 + 当前版本日志生成最终差异汇总
  • 统一串联过程留痕、成熟度管理、交付边界确认与开发关注重点
产出
  • looply-{模块名}-变更日志.md
  • V{上一版本}-V{本次版本}-核心差异汇总.md
沉淀规范
  • 需求迭代版本管理 skill~/.claude/skills/req-versioning/SKILL.md
核心原则
  • 基于实际交付文件对比：不基于假设、记忆或中间迭代版本
  • 删除功能判断：只有上一版包含、本次删除的才算"删除功能"
  • 新增功能完整性：列出所有业务功能模块，不只是交互细节
  • 交付包命名：交付开发 V{版本号}-{日期}（V1.1 起统一）
