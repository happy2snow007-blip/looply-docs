# 登录注册模块需求优先级

## 1. 功能清单汇总

从竞品对标、合规扫描、服务商选型三份调研中提取的全部功能点，去重分类后共 **33 项**。

### 1.1 注册相关（11 项）

| # | 功能项 | 来源 | 说明 |
|---|--------|------|------|
| R1 | 邮箱注册 | 竞品标配 | 邮箱 + 密码，行业标准注册方式 |
| R2 | Google OAuth 注册/登录 | 竞品标配 + 合规 | 6/6 竞品支持（Fashionphile 除外） |

| R4 | Facebook 注册/登录 | 竞品标配 | 6/6 竞品支持（Fashionphile 除外） |
| R5 | Facebook 无邮箱注册容错 | 合规（Facebook 数据策略） | 用户可能未绑定邮箱或拒绝共享，需允许无邮箱注册 |

## 2. RICE 评估

### 评分标准

| 维度 | 分值范围 | 说明 |
|------|---------|------|
| Reach（触达） | 1-5 | 影响多少用户。5=所有用户，3=大部分用户，1=极少数用户 |
| Impact（影响） | 1-5 | 对核心指标的影响。5=不做就不能上线，3=显著提升体验，1=锦上添花 |
| Confidence（确信度） | 1-5 | 方案确定性。5=行业标准/法规要求，3=有参考但需适配，1=探索性 |
| Effort（工作量） | 1-5 | 开发成本。1=极低（<1天），3=中等（3-5天），5=极高（>2周） |
| RICE 分 | 计算公式 | (Reach × Impact × Confidence) / Effort |

## 3. MVP / V2 / V3 划分建议

基于合规强制性、竞品标配程度、用户核心路径覆盖，给出初步划分建议。**Reach 和 Impact 确认后可调整排序。**

### MVP（必须上线，不做就不能发布）

合规红线 + 核心注册登录路径，共 **21 项**：

| 分类 | 功能项 | 理由 |
|------|--------|------|
| 注册 | R1 邮箱注册 | 核心注册方式 |
| 注册 | R2 Google OAuth | 行业标配，降低注册摩擦 |
| 注册 | R3 Apple Sign In | App Store 强制要求 |
| 注册 | R4 Facebook 登录 | 行业标配 |
| 注册 | R5 Facebook 无邮箱容错 | 不做会阻断部分用户注册 |
| 注册 | R6 Apple relay 邮箱 | 不做 Apple 用户收不到邮件 |
| 注册 | R7 邮箱验证 | 确保邮箱有效，行业标配 |
| 注册 | R8 年龄筛查（18+） | COPPA 合规红线 |
| 注册 | R9 隐私政策+服务条款 | CCPA + FTC 合规红线 |
| 注册 | R10 营销邮件 opt-in | CAN-SPAM 合规红线，实现成本极低 |
| 注册 | R11 国家/地区选择 | 影响价格体系/合规策略/数据分区，竞品标配 |
| 登录 | L1 邮箱+密码登录 | 核心登录方式 |
| 登录 | L2 三方 OAuth 登录 | 复用注册 OAuth |
| 登录 | L3 记住登录状态 | 基础体验，竞品标配 |
| 登录 | L6 人机验证 | 防机器人注册/暴力破解 |
| 安全 | S1 密码哈希存储 | FTC 合规红线 |
| 安全 | S2 HTTPS 全站加密 | FTC 合规红线 |
| 安全 | S3 忘记密码 | 核心功能，竞品标配 |
| 安全 | S6 密码策略（NIST） | 密码校验基础规则 |
| 账号 | A1 账号注销+数据删除 | CCPA 合规红线 |
| 账号 | A3 退出登录 | 基础功能 |

### V2（上线后尽快补齐，提升安全和体验）

安全增强 + 行业最佳实践，共 **7 项**：

| 分类 | 功能项 | 理由 |
|------|--------|------|
| 安全 | S4 密码修改 | 账号安全基础功能 |
| 安全 | S5 密码泄露库检查 | NIST 建议，HIBP 接入成本极低 |
| 登录 | L4 新设备登录验证 | 安全增强，Mercari/Vinted 已有 |
| 登录 | L5 登录失败锁定 | 防暴力破解，行业标准 |
| 风控 | F1 设备指纹采集 | 新设备验证的前置依赖 |
| 风控 | F2 IP 地理定位 | 异常登录检测辅助（注：国家自动检测能力已在 MVP R11 中实现） |
| 风控 | F3 风控日志记录 | 审计追踪，安全事件排查 |

### V3（中长期完善，锦上添花）

高级功能 + 低频场景，共 **5 项**：

| 分类 | 功能项 | 理由 |
|------|--------|------|
| 账号 | A2 注销冷静期 | 体验优化，非合规强制 |
| 账号 | A4 并发会话管理 | NIST 建议，用户量大时更有价值 |
| 风控 | F4 数据泄露通知机制 | 合规要求但属于安全体系建设，非登录注册模块独立完成 |
| 营销 | M1 营销邮件退订 | CAN-SPAM 要求，跟随营销邮件功能上线时一起做，初期无营销邮件则不需要 |
| 营销 | M2 隐私权行使入口 | CCPA 要求，可先用客服邮箱兜底，后续建自助入口 |
