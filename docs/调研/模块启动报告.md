# 登录注册模块启动报告

## 1. 模块定位与目标

looply 二手电商平台的用户入口模块，覆盖注册、登录、密码管理、账号安全、账号注销全链路。

- 目标市场：美国（前期），后续可能扩展欧洲
- 用户角色：统一账号，买家/卖家/商家角色由各业务域管理，不在本模块区分
- 核心目标：低摩擦注册转化 + 合规达标 + 基础安全保障

---

## 2. 竞品关键发现

1. **邮箱 + Google + Apple + Facebook 是行业标配**：6 家竞品中 5 家支持四件套（Fashionphile 仅邮箱，用户明确抱怨不便）
2. **注册信息越少越好**：Mercari 和 Back Market 只要邮箱+密码，注册转化最高；Poshmark 要姓名+性别+用户名，门槛最高
3. **三方登录按钮放在邮箱注册上方**：行业共识，优先引导快速注册
4. **交易时才要手机号**：所有竞品注册阶段都不强制手机号，首次交易时引导绑定
5. **安全与体验的平衡**：Mercari 的 SMS 2FA 安全但用户抱怨频繁，风控驱动的按需验证（正常不打扰，异常才触发）是更好的策略

## 3. 合规红线（必须遵守）

| # | 要求 | 法规依据 |
|---|------|---------|
| 1 | 密码哈希存储（Argon2id/bcrypt + 盐值） | FTC Act §5 |
| 2 | HTTPS 全站加密 | FTC Act §5 |
| 3 | 注册时展示隐私政策 + 服务条款（clickwrap） | CCPA + FTC |
| 4 | 年龄筛查（18+ 声明） | COPPA |
| 5 | 账号注销 + 数据删除通道 | CCPA |
| 6 | Apple Sign In 必须支持 | App Store 审核 |
| 7 | Apple relay 邮箱必须能收发 | Apple 要求 |
| 8 | Facebook 无邮箱注册容错 | Facebook 数据策略 |
| 9 | 营销邮件 opt-in（不默认勾选） | CAN-SPAM |
| 10 | 数据泄露通知义务 | CCPA + 各州法 |

特别注意：德克萨斯和蒙大拿隐私法无收入门槛，looply 初期规模小也适用。建议以 CCPA/CPRA 为基准设计，一套方案覆盖全美。

## 4. 技术依赖与选型建议

| 外部能力 | 建议方案 | 初期月成本 | 说明 |
|---------|---------|----------|------|
| 事务性邮件 | Amazon SES | ~$0→$10 | 成本最低，需自行管理送达率 |
| 人机验证 | Cloudflare Turnstile | $0 | 免费无限制，无感验证，隐私合规优势 |
| 设备指纹 | 自建（基础信息哈希） | $0 | 初期够用，后期可切 Fingerprint Pro |
| 密码泄露检查 | HIBP k-Anonymity API | $0 | 免费，NIST 推荐方案 |
| IP 地理定位 | MaxMind GeoLite2 | $0 | 国家级准确率 99.5% |

以上为调研建议，最终方案需在开发阶段与开发团队一起评估后确定。

## 5. MVP 功能清单（21 项）

### 注册（11 项）
- 邮箱注册（邮箱+密码）
- Google OAuth 注册/登录
- Apple Sign In 注册/登录
- Facebook 注册/登录
- Facebook 无邮箱注册容错
- Apple relay 邮箱支持
- 邮箱验证（注册后验证链接）
- 年龄筛查（18+ 声明）
- 隐私政策 + 服务条款展示（clickwrap）
- 营销邮件 opt-in（不默认勾选）
- 国家/地区选择（IP 自动检测 + 手动修改）

### 登录（4 项）
- 邮箱 + 密码登录
- 三方 OAuth 登录（复用注册）
- 记住登录状态（Token 持久化）
- 人机验证（Cloudflare Turnstile 或同类）

### 安全（4 项）
- 密码哈希存储（Argon2id/bcrypt）
- HTTPS 全站加密
- 忘记密码（邮件重置链接）
- 密码策略（NIST：≥8字符，不强制复杂度）

### 账号（2 项）
- 账号注销 + 数据删除
- 退出登录
