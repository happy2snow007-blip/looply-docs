# 登录注册模块服务商选型

## 选型范围

登录注册模块涉及的外部服务依赖：

| 序号 | 外部能力 | 用途 | 优先级 |
|------|---------|------|--------|
| 1 | 事务性邮件服务 | 验证码邮件、密码重置邮件、安全通知 | 必须 |
| 2 | 人机验证（CAPTCHA） | 注册/登录防机器人、暴力破解防护 | 必须 |
| 3 | 设备指纹 | 新设备识别、风控辅助、异常登录检测 | 建议 |
| 4 | 密码泄露库检查 | 注册/改密时检查密码是否已泄露 | 建议 |
| 5 | IP 地理定位 | 自动检测用户国家/地区、异常登录地检测 | 建议 |

**不在选型范围**：
- OAuth 提供商（Google/Apple/Facebook）：由合规要求和竞品标准决定，无替代选项
- 短信服务：当前阶段手机号不作为注册入口，后续绑定时再选型

## 1. 事务性邮件服务

### 1.1 候选方案

| 对比维度 | Amazon SES | SendGrid (Twilio) | Mailgun (Sinch) |
|---------|-----------|-------------------|-----------------|
| **定位** | 基础设施级邮件服务 | 全功能邮件平台 | 开发者友好邮件平台 |
| **免费额度** | EC2 实例发送：62,000封/月免费 | 100封/天（永久免费） | 试用期 5,000封/月 |
| **基础定价** | $0.10/1,000封（极低） | Essentials $19.95/月（50K封） | Foundation $35/月（50K封） |
| **10万封/月成本** | ~$10 | Pro $89.95/月 | Scale $90/月 |
| **API 质量** | REST + SMTP，文档完善 | REST + SMTP，SDK 丰富（7种语言） | REST + SMTP，SDK 丰富 |
| **送达率管理** | 需自行管理（IP 预热、退信处理） | 内置送达率优化 + 专家支持 | 内置送达率优化 |
| **Apple relay 支持** | 支持（需配置 SPF/DKIM） | 支持（需配置 SPF/DKIM） | 支持（需配置 SPF/DKIM） |
| **专用 IP** | 需额外申请 | Pro 及以上包含 | Scale 及以上包含 |
| **模板引擎** | 基础（SES Templates） | 强大（动态模板 + 可视化编辑器） | 强大（模板 + 变量） |
| **Webhook/事件追踪** | SNS 通知（需额外配置） | 内置（打开/点击/退信/投诉） | 内置（打开/点击/退信/投诉） |
| **接入复杂度** | 中（需熟悉 AWS 生态） | 低（独立服务，快速接入） | 低（独立服务，快速接入） |
| **美国市场适配** | AWS 美国基础设施成熟 | Twilio 旗下，美国市场主力 | 美国市场覆盖良好 |
| **合规认证** | SOC 2, ISO 27001 | SOC 2, ISO 27001, GDPR | SOC 2, GDPR |

### 1.2 推荐

**推荐方案：Amazon SES（首选）+ SendGrid（备选）**

理由：
- **成本优势明显**：SES 单价 $0.10/千封，是 SendGrid/Mailgun 的 1/5~1/10。登录注册模块邮件量大（验证码、密码重置、安全通知），成本差异会随用户增长放大
- **looply 大概率用 AWS**：如果后端部署在 AWS，SES 与 EC2/Lambda 天然集成，62K封/月免费额度足够初期使用
- **送达率需自行管理**：SES 的劣势是送达率需要自己做 IP 预热和退信处理，但登录注册邮件是事务性邮件（非营销），送达率问题相对小
- **SendGrid 作为备选**：如果团队不熟悉 AWS 或需要更好的模板编辑器和送达率管理，SendGrid Pro 是成熟方案

**Apple relay 邮箱注意事项**（三个方案都需要）：
- 在 Apple Developer 后台注册发件域名
- 配置 SPF + DKIM 记录
- 测试向 @privaterelay.appleid.com 地址发送邮件的送达率

## 2. 人机验证（CAPTCHA）

### 2.1 候选方案

| 对比维度 | Cloudflare Turnstile | Google reCAPTCHA v3 | hCaptcha |
|---------|---------------------|--------------------|---------|
| **定位** | 隐私友好的无感验证 | 最广泛使用的验证方案 | 隐私优先的验证方案 |
| **免费额度** | 无限（免费方案无请求限制） | 100万次/月免费 | 100万次/月免费 |
| **付费定价** | Enterprise 需联系销售 | Enterprise $8/千次评估 | Pro $99/月起 |
| **用户体验** | 无感验证（无需交互） | 无感评分（v3），可能弹出挑战（v2 fallback） | 图片挑战为主，有无感模式 |
| **隐私合规** | 不追踪用户，GDPR 友好 | 收集用户数据用于 Google 广告生态 | 不追踪用户，GDPR 友好 |
| **CCPA 影响** | 低风险 | 需在隐私政策中披露 Google 数据收集 | 低风险 |
| **接入复杂度** | 低（简单 JS + 后端验证） | 低（JS + 后端验证，文档成熟） | 低（JS + 后端验证） |
| **Bot 检测能力** | 强（Cloudflare 网络数据支撑） | 强（Google 搜索数据支撑） | 中-强 |
| **无障碍性** | 好（无需视觉交互） | 中（v3 好，v2 挑战对视障用户不友好） | 中（图片挑战有无障碍替代） |
| **是否需要 Cloudflare CDN** | 不需要（独立使用） | 不需要 | 不需要 |
| **美国市场适配** | 广泛使用 | 最广泛使用 | 使用增长中 |

### 2.2 推荐

**推荐方案：Cloudflare Turnstile**

理由：
- **免费且无请求限制**：对初创项目最友好，不用担心超额
- **用户体验最好**：完全无感验证，用户无需点击"我不是机器人"或做图片挑战，注册/登录流程零摩擦
- **隐私合规优势**：不追踪用户行为，不向第三方共享数据，CCPA/GDPR 合规压力最小。reCAPTCHA 会收集数据用于 Google 广告生态，需要在隐私政策中额外披露
- **不依赖 Cloudflare CDN**：可以独立使用，不要求网站接入 Cloudflare
- **Bot 检测能力强**：背靠 Cloudflare 全球网络数据，检测准确率高

**降级方案**：如果 Turnstile 在某些场景检测不准（如高级爬虫），可在高风险操作（如连续登录失败后）叠加 reCAPTCHA v2 挑战作为二级验证。

## 3. 设备指纹

### 3.1 候选方案

| 对比维度 | Fingerprint Pro (fingerprint.com) | 自建方案（开源库） | Cloudflare Bot Management |
|---------|----------------------------------|-------------------|--------------------------|
| **定位** | 专业设备识别 SaaS | 基于开源库自行实现 | CDN 级 Bot 管理 |
| **免费额度** | 20,000次 API 调用/月 | 免费（开源） | 需 Cloudflare Enterprise |
| **付费定价** | Pro $200/月起（100K 调用） | 仅服务器成本 | 需联系销售 |
| **识别准确率** | 99.5%（官方数据） | 60-80%（浏览器指纹易被伪造） | 高（但侧重 Bot 而非设备识别） |
| **跨浏览器识别** | 支持 | 不支持 | 部分支持 |
| **隐私合规** | GDPR/CCPA 合规，提供数据处理协议 | 自行负责合规 | Cloudflare 提供合规支持 |
| **接入复杂度** | 低（JS SDK + REST API） | 高（需自行开发+维护） | 中（需 Cloudflare Enterprise） |
| **适用场景** | 精确设备识别、账号安全 | 基础风控辅助 | 全站 Bot 防护 |

### 3.2 推荐

**推荐方案：初期自建 + 后期评估 Fingerprint Pro**

理由：
- **初期用自建方案够用**：登录注册模块的设备指纹主要用于"新设备登录通知"和基础风控，不需要 99.5% 的识别准确率。收集 User-Agent + 屏幕分辨率 + 时区 + 语言 等基础信息做哈希，足够区分大部分设备
- **成本控制**：Fingerprint Pro $200/月起，对初创项目偏贵。自建方案零成本
- **后期升级路径**：当用户量增长、欺诈风险上升时，再接入 Fingerprint Pro 提升准确率。API 接口设计时预留抽象层，方便后续切换
- **不建议 Cloudflare Bot Management**：它侧重全站 Bot 防护而非设备识别，且需要 Enterprise 套餐，杀鸡用牛刀

## 4. 密码泄露库检查

### 4.1 候选方案

| 对比维度 | Have I Been Pwned (HIBP) API | Enzoic | 自建布隆过滤器 |
|---------|----------------------------|--------|--------------|
| **定位** | 最知名的泄露密码检查服务 | 企业级凭证安全 | 本地离线检查 |
| **免费额度** | k-Anonymity API 完全免费 | 试用期后付费 | 免费（开源数据） |
| **付费定价** | 免费（API），企业版需订阅 | 需联系销售 | 仅存储和计算成本 |
| **数据库规模** | 10亿+ 泄露密码 | 数十亿凭证 | 取决于导入的数据集 |
| **隐私保护** | k-Anonymity（只发送哈希前5位） | 类似 k-Anonymity | 完全本地，无网络请求 |
| **接入复杂度** | 极低（REST API，一个接口） | 低（SDK 支持） | 高（需下载数据集+构建过滤器） |
| **延迟** | 低（CDN 加速） | 低 | 极低（本地查询） |
| **数据更新** | 持续更新 | 持续更新 | 需定期手动更新 |

### 4.2 推荐

**推荐方案：Have I Been Pwned (HIBP) k-Anonymity API**

理由：
- **完全免费**：k-Anonymity API 无调用限制，无需付费
- **接入极简**：注册/改密时，将密码 SHA-1 哈希的前 5 位发送给 API，返回匹配的哈希后缀列表，本地比对。一个 HTTP 请求搞定
- **隐私安全**：k-Anonymity 模型确保 HIBP 永远不知道用户的完整密码哈希
- **行业标准**：NIST 800-63B 推荐检查密码是否在已泄露库中，HIBP 是最广泛引用的实现方案
- **数据库最全**：10亿+ 泄露密码，持续更新

## 5. IP 地理定位

### 5.1 候选方案

| 对比维度 | MaxMind GeoLite2 | ipinfo.io | Cloudflare（附带功能） |
|---------|-----------------|-----------|---------------------|
| **定位** | 离线 IP 地理数据库 | 在线 IP 信息 API | CDN 附带的地理信息 |
| **免费额度** | 免费（GeoLite2 数据库） | 50,000次/月免费 | Cloudflare 用户免费 |
| **付费定价** | GeoIP2 $25-$300/月 | $99/月起（150K 次） | 包含在 Cloudflare 套餐中 |
| **准确度（国家级）** | 99.5% | 99%+ | 99%+ |
| **准确度（城市级）** | 80%+ | 85%+ | 不提供 |
| **数据更新** | 每周更新 | 实时 | 实时 |
| **接入方式** | 本地数据库查询（无网络延迟） | REST API | HTTP Header（CF-IPCountry） |
| **接入复杂度** | 中（需下载数据库+定期更新） | 低（REST API） | 极低（读 Header） |

### 5.2 推荐

**推荐方案：MaxMind GeoLite2（首选）或 Cloudflare Header（如已用 Cloudflare）**

理由：
- **登录注册只需国家级精度**：用于自动检测用户所在国家/地区，GeoLite2 国家级准确率 99.5% 足够
- **零成本 + 零延迟**：GeoLite2 免费，本地查询无网络延迟
- **如果已用 Cloudflare**：直接读 `CF-IPCountry` Header，零接入成本，零额外请求
- **ipinfo.io 作为补充**：如果需要更详细的 IP 信息（如 ASN、VPN 检测），ipinfo.io 免费额度足够初期使用
