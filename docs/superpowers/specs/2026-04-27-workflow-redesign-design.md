# PM AI 工作流页面 Redesign 设计方案

## 目标

重新设计 workflow.html，体现 PM 核心 AI 工作流程，方便向新人和外部门人员介绍。

## 受众

- 新入职团队成员
- 外部门协作人员（开发、测试、运营等）

## 页面结构

### 区域 1：Hero 区

- 紫色渐变背景，沿用现有品牌风格
- 标题："Looply PM AI 工作流"
- 副标题："从调研到交付，7 个核心 Skill 驱动的产品工作台"
- 导航链接：返回文档中心（index.html）、Skills 明细（skills-hub-hybrid.html）

### 区域 2：主流程时间线

垂直时间线，3 个阶段，每个阶段是一个大卡片，内含 skill 子卡片。

#### 阶段 1：产品架构（绿色色条）

包含 2 个 skill 子卡片：

1. **产品架构图** `/product-architecture-diagram`
   - 工作方法：读调研产出，定义模块边界（用户层/核心能力/外围系统）；Python 数据驱动生成 SVG，迭代只改数据不碰坐标
   - 产出：SVG 架构图（数据驱动 Python 生成）
   - 沉淀规范：架构图 skill、架构图绘制方法 memory

2. **实体关系图** `/er-diagram-generation`
   - 工作方法：从架构图提取实体，逐实体定义字段、类型、约束、枚举；HTML 双视图（产品视图+技术视图）
   - 产出：HTML 双视图（产品视图+技术视图）
   - 沉淀规范：ER图 skill

#### 阶段 2：需求设计（蓝色色条）

包含 4 个 skill 子卡片：

1. **系统流程图** `/swimlane-diagram-generation`
   - 工作方法：按用户入口划分段落，每段包含完整技术链路；engine.py 数据驱动生成 SVG
   - 产出：SVG 泳道图（engine.py 数据驱动）
   - 沉淀规范：泳道图 skill

2. **UI 设计稿** `/ui-design-workflow`
   - 工作方法：基于流程图逐页面设计，先 PC 后 H5，遵循品牌视觉规范；引用 looply-design.pen 组件库，新组件提取回组件库（双更新）；每轮出图后跑评估检查清单，用户确认后定稿
   - 产出：PC 端 .pen 设计文件、H5 端 .pen 设计文件、组件库更新
   - 沉淀规范：UI 出图流程 skill、品牌视觉规范、设计系统（组件库+Tokens）、品牌上下文

3. **后台原型** `/admin-prototype`
   - 工作方法：管理后台模块进入原型阶段时统一使用 skill；统一复用 logo 资源与后台 UI 规范；页面少时直接套用骨架，页面多时复用生成模板批量出原型；功能删除或裁剪时统一使用维护脚本
   - 产出：后台 HTML 单文件原型、原型生成脚本（按需）、功能维护配置 JSON（按需）、统一 logo 与 UI 规范引用
   - 沉淀规范：后台原型 skill、后台 UI 规范、静态 CSS 模板、通用生成模板、统一维护脚本、品牌 logo 资源

4. **PRD** `/prd-writing`
   - 工作方法：动手前交叉对齐所有输入源，列出覆盖差异；按页面/功能逐章撰写，每章关联 PC+H5 设计稿；写完后路径完整性检查+设计稿交叉校验
   - 产出：Word(.docx) 带自动目录、Markdown(.md) 在线阅读版
   - 沉淀规范：PRD 写作 skill、PRD 写作规范

#### 阶段 3：版本交付（紫色色条）

包含 1 个 skill 子卡片：

1. **需求迭代版本管理** `/req-versioning`
   - 工作方法：模式 A 持续记录进行中版本；模式 B 交付收口生成差异汇总；统一串联过程留痕、成熟度管理、交付边界确认与开发关注重点
   - 产出：变更日志、核心差异汇总
   - 沉淀规范：需求迭代版本管理 skill

### 区域 3：持续迭代补充说明

轻量区块，虚线边框或浅灰背景，与主流程视觉区分：

- 文案说明：需求迭代时修改的仍是阶段 1-2 的产出文件，/req-versioning 自动记录变更，交付前生成差异汇总
- 两个小标签并排：模式 A（迭代留痕）→ 模式 B（交付收口）
- 不展开详细内容，整体占页面约 10% 篇幅

### 区域 4：Footer

- "查看所有 Skill 明细"链接（→ skills-hub-hybrid.html）
- 版权信息：Looply &copy; 2026 · 内部文档，请勿外传

## 视觉设计

- **配色**：沿用现有品牌色系，阶段色条（绿/蓝/紫）
- **卡片样式**：沿用现有 flow-node 圆角卡片 + 左侧色条
- **时间线**：阶段之间用箭头连接
- **响应式**：桌面端阶段 2 的 4 个 skill 可 2x2 网格，移动端全部纵向堆叠
- **密码墙**：保留现有 auth-wall 机制

## 数据来源

所有 skill 内容复用现有 workflow.html 中的文案，不新增不删减。7 个星标 skill 对应 skills-hub-hybrid.html 中的 coreProductSkills 数组。

## 不包含

- 非星标 skill（module-kickoff、competitor-analysis 等）
- 进度看板视图（原有但已不再需要）
- 模块卡片式看板视图（原有但已不再需要）
