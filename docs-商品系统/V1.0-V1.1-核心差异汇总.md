## 一、删除的功能

### 1.1 属性分组管理（已删除）
**删除原因**：简化属性管理体系，减少复杂度

**V1.0 状态**：✅ 包含（PRD v0.2 第 14 行明确列出"属性分组"）

**删除内容**：
- 菜单项：属性分组管理一级菜单
- 页面：属性分组列表页、属性分组新建页、属性分组编辑页（3 个页面）
- 数据表：attribute_group 表及相关外键

**影响**：
- 属性定义直接挂载在类目下，不再通过分组管理
- 属性作用域配置更加直接

## 二、新增的功能

### 2.1 质检报告模板管理（新增）
**新增原因**：标准化质检流程，支持按类目配置质检模板

**新增内容**：
- **质检模板管理页面**：
  - 质检模板列表页：展示所有质检模板，支持搜索和筛选
  - 质检模板新建/编辑页：配置模板基础信息和结构（维度 → 质检项）
  - 结果选项配置弹窗：为质检项配置可选结果（如：无、轻微、明显、严重）

- **商品创建集成**：
  - 根据类目自动加载对应质检模板
  - 动态生成质检结果填写表单（按维度分组）
  - 无模板警告态：类目未配置模板时的提示和引导

- **商品详情展示**：
  - 按维度分组展示质检结果
  - 支持查看完整质检报告

**功能特点**：
- 一个类目对应一个质检模板
- 模板结构：维度（如外观、五金、配件）→ 质检项（如划痕、磨损、完整性）
- 质检项支持枚举选择和文本输入两种结果类型

### 2.2 点击效果示意（新增）
**新增原因**：避免按钮点击无反应，提升原型可用性

**新增内容**：
- **P0 高频操作**（4 个弹窗）：
  1. 添加类目弹窗：品牌/系列新建和编辑页面，树形结构展示类目层级，支持多选
  2. 添加允许单位弹窗：属性定义新建和编辑页面，表格展示单位列表，支持多选
  3. 添加选项弹窗：属性定义新建和编辑页面，表单输入选项值、排序、状态
  4. 选择 SPU 弹窗：创建实物商品页面，表格展示 SPU 列表，支持搜索和筛选

- **P1 中频操作**（5 个弹窗）：
  1. 配置尺码体系弹窗：属性作用域新建和编辑页面，展示可用尺码体系列表
  2. 新增质检项弹窗：SKU 编辑页面，表单输入质检项信息
  3. 新增质检维度弹窗：SKU 编辑页面，表单输入维度信息
  4. 新增质检选项弹窗：SKU 编辑页面，表单输入选项信息
  5. 添加渠道弹窗：商品编辑和渠道商品编辑页面，表格展示渠道列表

- **属性作用域页面**（3 个弹窗）：
  1. 添加属性弹窗：从全局属性定义中选择
  2. 配置选项弹窗：勾选当前作用域允许使用的选项
  3. 配置单位弹窗：选择默认展示单位和允许使用的单位

**技术实现**：
- 所有交互按钮已添加 onclick 事件
- 弹窗设计统一遵循 Looply 后台 UI 规范
- JavaScript 控制函数已添加到页面底部

### 2.3 空态规范（新增）
**新增原因**：规范列表页的各种状态展示

**新增内容**：
- **零数据引导态**：列表无任何数据时，隐藏搜索/筛选栏，展示居中引导区（图标 + 标题"暂无{实体名称}" + 描述 + 主按钮"新增{实体名称}"）
- **筛选无匹配态**：搜索或筛选后无结果时，保留搜索/筛选栏，表格区域替换为居中提示（图标 + 标题"未找到匹配结果" + 描述 + 按钮"清除筛选条件"）
- **网络异常态**：请求失败时，保留搜索/筛选栏，表格区域替换为居中提示（错误图标 + 标题"加载失败" + 描述 + 主按钮"重试"）

**视觉规范**：详见《Looply 后台原型 UI 规范》6.13 节

### 2.4 变更日志（新增）
**新增原因**：记录版本演进，方便开发团队了解变更历史

**文件**：looply-商品系统-变更日志.md（25 KB）

**内容**：

- 版本号规则、功能成熟度标记、交付流程说明

## 七、常见问题

**Q1：为什么删除属性分组？**
A：简化属性管理体系，避免过度设计。当前阶段属性直接挂载在类目下即可满足需求。

**Q2：点击效果示意是否需要后端支持？**
A：不需要。点击效果示意仅为前端交互，后端接口无变化。

**Q3：空态规范是否所有列表页都要实现？**
A：是的。所有列表页都需要实现零数据引导态、筛选无匹配态、网络异常态三种状态。

**Q4：实体关系图和产品架构图为什么没变？**
A：本次变更主要是删除功能和补充交互细节，核心数据模型和架构未调整。

**Q5：商家种子数据为什么没包含在 V1.1？**
A：V1.0 已交付，V1.1 为增量交付，不重复包含已交付的种子数据。

**Q6：派生维度为什么没有列在删除功能中？**
A：派生维度在 V1.0 版本中就没有包含，不属于本次删除的功能。
