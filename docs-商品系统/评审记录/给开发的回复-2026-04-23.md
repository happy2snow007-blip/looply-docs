以下为针对本轮反馈的逐条回复，基准版本为 **交付开发 20260420（原型 v14 + PRD v0.2）**，当前最新版本为 **原型 v26 + PRD v0.5**。

### 1. 关于尺码换算的多对多关系和争议区间

这个反馈部分成立，但当前版本的设计已经通过数据建模避免了大部分一对多问题。

**当前版本的处理方式：**

尺码值会按性别分开维护（如"男5码"、"女5码"作为两个独立的尺码值），因此不会因为性别差异导致一对多映射。大部分标准换算在性别区分后是确定的 1:1 关系。

对于少数边界情况的区间值/争议值（如某些品牌的 UK 5 可能对应 EU 38 或 38.5）：
- 运营选择最常用的映射值
- 通过备注字段说明争议情况
- 前台展示时加提示文案"尺码换算仅供参考，请以商品详情为准"

**后续版本按需覆盖的能力：**
- 多对多映射、区间值表达、置信度标注
- 争议规则的审核流程
- 换算方向元数据（单向/双向）

因此，这一条按"**当前版本 1:1 模型已满足需求，复杂场景后续版本按需覆盖**"处理。

---

### 2. 关于商品录入的跨步校验、自动存草稿、进度保存

这个反馈中提到的三项能力，部分已在当前版本覆盖。

**已覆盖的能力：**

1. **跨步校验**：PRD v0.5 已明确"未完成必填步骤：提示'请先完成步骤 N 的必填信息'"，Step 1 必须选择 SKU 后才能进入后续步骤。

2. **网络异常保护**：PRD v0.5 已明确"网络异常：提示'提交失败，请重试'，已填写信息不丢失"，前端在内存中保持表单状态，网络异常时不清空。

**当前版本不做的能力：**

自动存草稿（后端持久化 + 下次恢复）和进度保存指示属于用户体验优化，不在当前 MVP 范围内。当前版本先保证核心录入流程跑通，草稿机制后续版本按需补充。

因此，这一条按"**跨步校验和网络异常保护已覆盖，草稿自动保存后续版本按需覆盖**"处理。

---

### 3. 关于批量操作的交互细节

批量操作不在当前版本范围内。

PRD v0.5 版本规划中明确：批量上架属于 V2.0 范围。当前版本聚焦单个商品的完整操作链路。

原型中已有的批量操作按钮保留，作为后续版本的功能预留入口。当前版本开发时，这些按钮可以先做占位，点击时提示"该功能将在后续版本中提供"。

后续 V2.0 版本会完整设计批量操作能力，包括行选择交互、进度反馈、失败项清单与重试、中途取消等。

因此，这一条按"**当前版本不实现，V2.0 覆盖**"处理。

### 4. 关于操作日志缺失

这个反馈成立，**当前版本需要补充操作日志功能**。

二手奢侈品涉及高价值数据，操作可追溯性是基础能力，不应该延后。

**当前版本的处理方式：**

操作日志嵌入到各业务对象的详情页中，直接可见，不做成独立的审计日志模块。具体方案：

1. **展示位置**：各业务对象详情页底部新增"操作记录"区域（如商品详情页、SPU 详情页等）
2. **展示格式**：时间轴或列表，按时间倒序展示
3. **记录字段**：操作人、操作时间、操作类型、变更内容（变更前后值对比）
4. **覆盖的关键操作**：状态变更、价格修改、属性修改、上架/下架等

**后续行动：**
- ✅ PRD 补充操作日志功能说明（已完成：全局约定第 120 行）
- ✅ 原型补充操作日志的效果示意（已完成：v27 商品详情页、类目编辑页、品牌编辑页）

### 5. 关于停用级联的影响范围预览

这个反馈成立，当前版本已有基础防护，需要进一步增强。

**已覆盖的能力：**

PRD v0.5 中已有以下防护机制：
- 类目停用：弹窗确认"该类目下有 N 个 SPU，停用后相关 SPU 将无法正常使用，是否继续？"
- 渠道停用：弹窗确认"该渠道有 N 件商品在售，停用后这些商品将自动下架，是否继续？"
- 标签停用：如果标签正在被商品使用，需提示确认

**需要增强的能力：**

1. **影响范围预览列表**：当前只显示影响数量，没有详细列表。补充：弹窗中增加"查看详情"链接，展开受影响对象列表（显示 ID、名称、状态）
2. **品牌停用的级联规则**：当前 PRD 未明确说明停用品牌是否级联影响系列。补充：停用品牌时提示"该品牌下有 N 个系列，停用后系列将无法使用，是否继续？"

**后续行动：**
- ✅ PRD 补充影响范围预览列表的交互说明（已完成：全局约定第 121 行）
- ✅ PRD 补充品牌停用的级联规则（已完成：品牌编辑异常处理段落、类目编辑停用描述对齐）
- ✅ 原型补充停用影响范围预览弹窗示例（已完成：v27 impact-preview-dialog）
